Analytical Methods

Our analysis combines descriptive statistics, nonparametric hypothesis testing, survival modeling, and count regression to characterize and predict player career length.

Pipeline Overview

  1. Data collection: scrape and export player-season / roster records.
  2. Data cleaning: standardize league names, roles, and season mapping.
  3. Feature engineering: derive career length, debut era, and tier transitions.
  4. EDA: summarize distributions, missingness, and cohort patterns.
  5. Hypothesis tests: compare regions and transition outcomes.
  6. Modeling: train Cox and Negative Binomial models.

Methods Used

Method Purpose Where used
EDA (distribution + cohort summaries) Understand baseline structure and data quality `pages/eda.html` and EDA figures
Kruskal-Wallis + Mann-Whitney U Compare career lengths across leagues `pages/hypothesis3.html`
Tier transition cohort analysis Measure Tier 2 to Tier 1 promotion rates/timing `pages/hypothesis2.html`
Cox Proportional Hazards Estimate survival probability over seasons Inference API and `pages/predictor.html`
Negative Binomial Regression Predict expected career length (count outcome) Inference API and `pages/predictor.html`

Model Inputs and Outputs

Inference input features

  • League: LCK, LPL, LEC, LCS
  • Role: Top, Jungle, Mid, Bot, Support
  • Debut season: integer season index (S6-S15)

Inference outputs

  • Predicted expected career length (seasons)
  • Season-by-season survival probabilities
  • Median survival season (first point below 50%)
  • Confidence label and human-readable interpretation
